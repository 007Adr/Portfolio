Contexte de l'atelier

GLPI (Gestionnaire Libre de Parc Informatique) est une application web open source distribuée sous licence GPL qui permet aux entreprises de gérer leur système d'information : gestion des services (ITSM), assistance utilisateurs (Help Desk) et inventaire complet des ressources via OCS Inventory.

Cet atelier consiste à installer GLPI sur un serveur Debian existant qui héberge déjà OCS Inventory, à sécuriser l'installation, à configurer la synchronisation automatique entre les deux outils via un plugin dédié, puis à enrichir GLPI avec des plugins supplémentaires issus des catégories Tendances, Nouveaux, Populaires, Mise à jour et Tags.

Installation de GLPI sur le serveur OCS existant
Installation · Debian
Contexte

GLPI est installé sur la même VM Debian que OCS Inventory (NomEt-D12-OCS). Avant toute manipulation, un clone de sauvegarde NomEt-D12-OCS-SAVE est créé sur Proxmox pour pouvoir revenir à un état stable en cas de problème.

Ce que j'ai fait
  • Clonage de la VM OCS vers NomEt-D12-OCS-SAVE sur Proxmox avant toute modification
  • Installation de GLPI en suivant le tutoriel TUTORIEL-INSTALLER-GLPI-10.0.10-SUR-DEBIAN-12.pdf, en adaptant les étapes aux paquets déjà présents sur le serveur OCS
  • Vérification de la page d'accueil GLPI accessible via l'URL du serveur
✅ GLPI installé et accessible via navigateur
2
Résolution des 3 alertes de sécurité post-installation
Sécurité · Hardening
Problème

Après installation, la page d'accueil GLPI affiche trois alertes de sécurité à résoudre avant toute utilisation en production.

Les 3 alertes et leur résolution
  • Alerte 1 — Mot de passe par défaut : changement des mots de passe des comptes par défaut glpi, post-only, tech et normal depuis l'interface d'administration GLPI
  • Alerte 2 — Fichier install/install.php : suppression du fichier d'installation via la commande rm /var/www/html/glpi/install/install.php pour empêcher toute réinstallation accidentelle
  • Alerte 3 — Configuration du dossier racine : déplacement du dossier de données GLPI hors de la racine web (/var/www/html) et configuration du chemin dans downstream.php pour que les fichiers non publics ne soient pas accessibles directement via l'URL
✅ 3 alertes résolues — Installation sécurisée
Synchronisation avec OCS Inventory
3
Installation du plugin OCS Inventory NG & synchronisation
Plugin · OCS · Synchronisation
Problème

Bien qu'installés sur le même serveur, GLPI et OCS Inventory ne communiquent pas par défaut. Il faut installer un plugin GLPI qui permet l'importation automatique et la mise à jour des inventaires depuis OCS.

Solution mise en place

Téléchargement du plugin OCS Inventory NG depuis la section Ressources → Plugins du site GLPI. Installation dans le répertoire /var/www/html/glpi/plugins/ puis activation depuis l'interface d'administration GLPI.

Étapes de configuration
  • Téléchargement et décompression de l'archive du plugin dans glpi/plugins/
  • Activation du plugin dans Administration → Plugins → Installer puis Activer
  • Configuration de la connexion : renseigner le serveur OCS (localhost), la base de données ocsweb, l'utilisateur et le mot de passe
  • Lancement de l'import en masse depuis Configuration → Actions automatiques → OcsSync
  • Vérification de la remontée d'inventaire : les machines OCS apparaissent dans Parc → Ordinateurs
✅ Inventaire OCS synchronisé dans GLPI
